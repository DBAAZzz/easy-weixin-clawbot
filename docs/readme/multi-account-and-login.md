# 多账号与微信登录

## 多账号模型

这个项目把每个微信号都视为一个独立 Agent：

- 独立账号状态
- 独立会话历史
- 独立记忆
- 独立工具/技能运行上下文
- 独立模型配置

后台可以统一查看、管理和切换这些账号。

## 登录方式

- 在 Web 后台打开 `/login`
- 服务端启动扫码流程
- 页面展示二维码，微信扫码确认
- 登录成功后，账号进入运行时

如果同一个微信号重复登录，新的登录态会接管旧实例，旧账号会进入废弃态。

## 登录态存储

当前实现里，微信相关状态已经数据库化：

- 凭据密文保存在 PostgreSQL
- 同步状态保存在 PostgreSQL
- 服务重启后可从数据库恢复

这意味着：

- 同一个数据库上的服务实例不需要重新扫码才能读取已有凭据
- Docker 部署不再依赖宿主机本地状态目录

## 管理入口

- 账号列表：`/`
- 扫码接入：`/login`
- 会话详情：`/accounts/:accountId`
